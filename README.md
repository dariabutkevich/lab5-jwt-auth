N.5 Развертывание приложения с JWT-токенами. CI/CD.
Реализуйте аутентификацию с использованием JWT (возьмите готовый вариант с предыдущей лабораторной) с последующим тестированием и настройкой CI/CD:

Выполните сборку проекта:
Создайте приложение с аутентификацией с использованием JWT (возьмите из предыдущей работы).
Убедитесь что у вас есть база данных (PostgreSQL, MySQL, MongoDB, или др.) для хранения данных.

Выполните тестирование приложения:
Включите юнит-тесты для проверки отдельных компонентов приложения и интеграционные тесты для проверки взаимодействия между компонентами.
Убедитесь, что все тесты проходят успешно.

CI/CD настройка:
Выберите бесплатный сервис CI/CD, например Jenkins или GitLab CI/CD или др.
Зарегистрируйтесь на выбранном сервисе и создайте новый проект. Настройте сборку проекта и интеграцию с вашим репозиторием кода (GitHub или GitLab или др.). Добавьте шаги тестирования, чтобы автоматически запускать тесты при каждой сборке. Настройте автоматическое развертывание вашего приложения на выбранном сервисе, после успешного прохождения тестов. Убедитесь, что развернутое приложение работает корректно.
Что нужно продемонстрировать в результате:
Вы внесли изменения в код, сохраняете в git и делаете git push.
Выполнение автоматической сборки (CI/CD-система увидела изменения в новом коммите и выполняет сборку )
Выполнение автоматических юнит-тестов (после выполненной сборки проекта запускаются юнит-тесты)
Выполнение деплоя на тестовое окружение (автоматически разворачиваете код в тестовом окружении(staging), базу данных и конфигураций)
Выполнение дополнительного тестирования (автоматически запускаются интеграционные тесты)
Выполнение деплоя в продакшн (после всех тестов приложение готово к продакшн)
Система мониторинга (проверьте ключевые показатели: количество ошибок, нагрузка на сервер и т.д.)
Выполнение отката приложения при наличии проблем (если есть критические проблемы, выполните процедуру отката к предыдущей версии приложения)
